# ShareManager autonomous 교훈 아카이브

> autonomous.md v2.0~v3.7의 교훈 중 ShareManager에서 탄생한 것들의 원본 기록입니다.
> 범용화된 원칙은 `autonomous.md` v3.7에 반영되어 있습니다.

---

## v2.0 (2026-01-19): AEGIS Protocol 통합
- ShareManager의 7-Layer 검증 프레임워크를 autonomous에 통합
- 원래 ShareManager CLAUDE.md에 있던 AEGIS를 범용 프레임워크로 추상화

## v2.1~v2.2 (2026-01-19): 랄프 루프
- ShareManager 테스트/검증 반복에서 영감
- "놀이터 철학": 실패할 때마다 표지판(검증 규칙)을 세우는 패턴

## v2.5 (2026-02-04): autonomous 자동 커밋
- autonomous.md 개선 시 자동으로 git 반영하는 규칙 도입

## v2.6 (2026-02-05): 문서 동기화 시스템
- **교훈**: 코드만 커밋하고 기술표를 나중에 업데이트 → 불일치 발생
- **원칙**: 코드와 문서는 한 커밋에 (기술표 SSOT)
- **강제**: 커밋 전 기술표 업데이트 확인 체크리스트

## v2.7 (2026-02-05): 타임존 규칙
- **교훈**: `.isoformat()` 단독 사용 → 타임존 정보 없음 → 9시간 오차
- **원칙**: 모든 datetime API 응답에 타임존 정보 포함 필수
- ShareManager에서 40곳 이상 수정

## v2.8 (2026-02-06): Phase 0 강제 도입
- **교훈**: "기술표 참조" 규칙이 있었으나 강제성 없어 무시됨
- **원칙**: 규칙의 존재 ≠ 규칙 준수. 강제 메커니즘 필요
- Phase 0이 모든 작업의 진입점을 통제

## v2.9 (2026-02-06): 에이전트 결과 검증
- **교훈**: 에이전트가 `source_type='GENERATED_BY_COMPETITOR'` 보고 → 실제는 `'competitor_info'`
- **원칙**: 에이전트 보고값은 반드시 원본 소스와 대조 검증
- **교훈**: "1, 3은 필요가 없겠다" → 주어 불명확 → 자의적 해석 위험
- **원칙**: 모호한 발언 시 반드시 확인, 빠른 결론 도출 금지

## v3.7 (2026-02-15): 사용자 여정 일관성 원칙 (Phase 5.8) — 규칙 R

- **교훈**: 단일 API의 계산 로직만 확인하고 "정상" 판정 → 연결 페이지에서 불일치 발견
- **원칙**: "계산이 맞다 ≠ 시스템이 맞다"
  - 단일 함수의 기술적 정확성만 확인하고 "버그 아님" 판정하는 것은 단편적 사고
  - 같은 이름의 지표가 다른 곳에도 있는지 grep으로 확인 필수
  - 해당 화면에서 링크/버튼으로 이동 가능한 페이지의 데이터와 일치하는지 확인 필수
  - 모든 표시 지점의 데이터 소스가 동일하거나 일관되는지 확인
- **자가 점검**: "계산이 맞다고 결론 내리려 하고 있나? → 다른 페이지에서도 확인했나?"
- **적용 시점**: 사용자가 데이터 값의 정확성을 질문할 때
