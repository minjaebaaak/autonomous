Infinite Loop (Ralph Wiggum 모드) - 목표 달성까지 무한 반복.

복잡한 작업을 목표 달성까지 자동으로 반복 실행합니다.

## 동작 원리

```
[목표 설정] → [작업 수행] → [검증] → [실패?] → [분석] → [수정] → [재시도]
                                ↓                           ↑
                            [성공] → 완료                (최대 10회)
```

## 사용 시나리오

| 시나리오 | 목표 예시 |
|----------|----------|
| 기능 구현 | "사용자 프로필 페이지 구현" |
| 버그 수정 | "로그인 시 세션 만료 버그 해결" |
| 리팩토링 | "API 응답 구조 통일" |
| 테스트 통과 | "모든 테스트 케이스 통과" |

## 실행 단계

### 1. 목표 분석

```
입력: "버튼 클릭 시 API 호출 기능 추가"

분석 결과:
- 대상 파일: [파일 목록]
- 필요 작업: [작업 목록]
- 성공 기준: [검증 방법]
```

### 2. 이터레이션 실행

**각 이터레이션마다:**

1. 작업 수행
2. /feedback-loop 실행 (검증 + 자동 수정)
3. 목표 달성 여부 확인
4. 미달성 시 분석 후 재시도

### 3. 반복 제어

```
최대 반복 횟수: 10회
피드백 루프 내부: 각 3회 시도
총 최대 시도: 10 x 3 = 30회
```

### 4. 중단 조건

**즉시 중단:**
- 환경 설정 오류 (Node.js, pnpm 등)
- 네트워크 오류 (SSH, API 연결 실패)
- 파일 시스템 오류 (권한, 경로)

**사용자 확인 후 중단:**
- 10회 반복 초과
- 동일 에러 3회 연속 발생
- 목표 달성 불가 판단

## 주의사항

- 복잡한 작업은 작은 단위로 분해
- 각 이터레이션 결과를 TodoWrite로 추적
- 무한 루프 방지를 위한 10회 제한 엄수
- Stop 훅으로 완료/실패 시 알림 발송

## 수동 실행

```
/infinite-loop "목표 설명"
```

예시:
```
/infinite-loop "크롤링 결과 페이지에 필터 기능 추가"
/infinite-loop "DNA 매칭 정확도 90% 이상 달성"
```
